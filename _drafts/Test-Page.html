Understanding the HTML DOM

The early Web, JavaScript and the DOM

Date: June 1, 2016

Categories: JavaScript, HTML

Tags: DOM, internet, WWW history, APIs

Why Understand the DOM?

Understanding the HTML DOM is critical for using JavaScript to make dynamic Web pages. Potential employers often ask Front-End applicants to testify or display in some way how well they understand the DOM. This is because everything JavaScript does to a Web page is accomplished by interfacing with it. For the sake of clarification, I should mention that there are actually two DOMs defined by W3C (click the link for an article about the differences), one for XML documents and one for HTML documents. This is a brief introduction to the HTML DOM.

A Short History of Web

Although early work was done that led up to the final product, the World Wide Web as we know it was invented and implemented in the late eighties and early nineties. It was a new and unique type of communication over the 'new' internet, which preceded the Web by several years. The internet was already being used for newsgroups (Usenet), email, file transfers and a few other things, mostly for the government and a few schools. Distinct from the existing internet traffic, the WWW portion of the internet would use a completely new protocol that defined how this communication worked, HTTP. The definition of the Web you'll most commonly find online is essentially, "all of the documents on servers on the internet that are connected with HTTP."

The WWW was the product of a European scientist named Tim Berners-Lee. He designed it for researchers to share their work with each other. Therefore in the early days, the Web was primarily academic texts connected to each other over the nascent internet backbone, which then mostly ran between a few academic and government institutions.

These early documents on the Web were completely static pages that could only be read. This was quite useful for sharing academic research and government papers, but far short of what we know today as THE Web.

In 1993 the first broadly used graphical Web browser, Mosaic, was released. Mosaic brought multimedia--images, sounds and video--to regular users. Mosaic soon begat Netscape, released in 1994, and the Web began to grow in content and public access exponentially.

JavaScript was created by Netscape in 1995, originally to manage the then popular Java applets providing animation in Web pages. As use of the World Wide Web grew in the mid-nineties, the Netscape team recognized the need for a more interactive and engaging experience for users. They envisioned a programming language that would allow Web pages to be manipulated on demand. This new tool would be able to react to events anywhere in the Web page (like clicks from a growingly common mouse, with Microsoftâ€™s release of the graphical Windows 3.1 in 1992), and then through the browser make changes to the actual content of the Web page.

JavaScript was enlisted to fill this void. It became the means to allow browsers to make changes to the HTML that defined the Web page, based on user actions (such as allowing users to 'flip' through a series of images on the page). JavaScript brought the ability to make things change with 'click' and 'rollover' mouse events, make custom information requests from the Web server on behalf of the user, push out page updates from the page provider (like news tickers), and even submit forms of information back to Web sites.

Enter the DOM

DOM is bridgin it

In order for JavaScript to do this amazing magic it had to be able to communicate with the page through the browser. The solution for the this was the DOM or Document Object Model. The DOM provides access to all of the objects within a Web page's HTML. The DOM is a bridge between the JavaScript code and the Web page.

The DOM defines the nodes within a web page (note: throughout this article and generally with regard to the DOM, the terms node, element, document objects, node objects, or just objects are interchangeable). Everything in an HTML Web page is either a node object or a component of one.

What differentiates one node from another is that every object has a minimum of three things specified for it: properties that define it, methods that can access it, and events that can trigger a change to it like a mouse click, a keypress, or a page error. These together make each object on the page distinct from any other (unless it is an intentionally replicated object), and therefore individually actionable.

Examples of DOM nodes include paragraphs, headings, comments, images, lists, footers, divs, spans, meta tags, links, or anything that can be an element within the HTML document.

The DOM standard is now maintained by the W3C, like HTML, and is public, so other programming languages can also access it; however JavaScript is still the primary language used by developers to manipulate Web content in pages.

The DOM is ephemeral and is created as a page loads and the browser reads the HTML in it. Part of a browserâ€™s job is to read the Web page source and create the DOM for it. The DOM then builds the page for the browser to display and sets the parameters for everything that can happen on the page going forward.

An API

Enter my lair...

The DOM manages the page by providing an Application Programming Interface, or API, for the JavaScript code to connect to. An API is the 'public' side of a program. It is the DOM's API that JavaScript can access to modify the page. An API acts like a gate to a well-armoured castle. Like a computer port, it allows certain acceptable communications to pass through to the program.

So specifically, the API provides access to the public methods that the DOM provides to manipulate each element in the page. As mentioned previously, each element, depending on what it is, has certain things that can be done to it, such as changing the text content (known as innerHTML) of a paragraph or submitting a form when a button is clicked (the .onclick event).

Accessing the DOM

An Example:

  <!DOCTYPE html>
  <html>
  <body>

    <!-- HTML begins -->
    <p>This example uses the HTML DOM to assign an
       'onclick' event to a button element.</p>

    <!-- A simple, unstyled HTML button -->
    <button id='change_label'>Click me</button>

    <!-- The JavaScript -->
    <script>
      <!-- When the button with the id 'change_label'
      is clicked, it triggers the function named
      'myFunction' -->
      document.getElementById('change_label').onclick =
          function() {myFunction()};

      <!-- This is 'myFunction' -->
      function myFunction() {
          document.getElementById('change_label').innerHTML =
          'YOU CLICKED ME!';
      }
    </script>

    <p style='padding-top: 8px;'>
      Refresh the page to run this again!
    </p>

  </body>
  </html>

Output:

This example uses the HTML DOM to assign an "onclick" event to a button element.

Refresh the page to run this again!

This is a complete sample Web page (although for brevity not completely HTML5 complient) that displays a simple paragraph; note that everything inside the <script> and </script> tags is invisible to the user viewing the page. This is an explanation of the JavaScript:

  1. The script includes two sections, the first begins with document, this is the root-node of and top object in the DOM tree.
  2. .getElementById is a method that the DOM provides to manipulate any elements that include a defined id in the page, like the button element above.
  3. The script then provides the id to be affected: change_label.
  4. .onclick is the event that triggers whatever instructions follow.
  5. myFunction is called.
  6. In the second section of the script document.getElementById('change_label') all mean the same as the first section.
  7. .innerHTML is a property of all paragraph elements and refers to the text wrapped inside the element (e.g., <p>Any text</p>).
  8. And the assignment operator = then reassigns the text property of the button to â€˜YOU CLICKED ME!â€™.

All attributes of elements can be changed with JavaScript through the DOM: styles (appearance), content, behaviors (like animations), etc.

Events can be triggered by many things such as keypresses, button clicks, hovering over an element (like a tooltip that pops up when your mouse moves over a word or section of a page), the page loading (requiring no interaction from the user) and much more.

Another Example:

  <!DOCTYPE html>
  <html>
  <body>

    <!-- HTML begins -->
    <div

      <!-- Calling the script functions -->
      onmouseover='mOver(this)'
      onmouseout='mOut(this)'

      <!-- Formatting the div's display -->
      class='center-block'
      style='background-color: #D94A38;
             text-align: center;
             width: 250px;
             height: 30px;
             margin-bottom: 15px;
             padding: 20px 0 40px 0;'>

      <!-- The text to display -->
      <p>Move your mouse over me!</p>

      <!-- Inline script -->
      <script>
        function mOver(obj) {
          obj.innerHTML = 'Thanks!'
        }
        function mOut(obj) {
          obj.innerHTML = 'You left me!'
        }
      </script>

    </div>
  </body>
  </html>

Output:

Move your mouse over me!

This example begins with a formatted (styled) div containing a paragraph element with the text 'Move your mouse over me!'. The script section contains two actions (i.e., methods) that can be executed to change the div's properties, mOver(obj) and mOut(obj). These are tied to two events defined in the HTML at the beginning of the div, onmouseover and onmouseout.

When the mouse cursor moves over (i.e., 'into') the div:

  1. onmouseover triggers the method mOver(obj) in the script.
  2. mOver(obj) re-assigns the text property of the paragraph element <p> to a new value.
  3. When the mouse moves out of the element, onmouseout triggers another event from the function mOut(obj), again reassigning the element's text (.innerHTML) property.

This example displays one of a number of ways to implement the common practice of modifying elements on a page when the mouse moves across them. For example, most people are familiar with a button changing (getting a glow or getting a bit larger, etc.) when your mouse passes over it.

These examples show how JavaScript is used to access the DOM and change the content of the page. JavaScript is a programming language with wide capabilities today, but it is at its best when creating 'magic' on Web pages.
